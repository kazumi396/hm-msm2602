# コーディングルール差分一覧

本リポジトリは、エンヴィデザイン様のコーディングルールを拝見し、命名規則や設計思想を意識して再実装した検証用プロジェクトです。

---

## 1. HTMLバージョン

| ルール                          | 本プロジェクトの対応                              | 差分     |
| ------------------------------- | ------------------------------------------------- | -------- |
| HTML Living Standard を使用する | `<!doctype html>` および HTML5 の要素・属性を使用 | **一致** |

---

## 2. Webフォント

| ルール                                                                                 | 本プロジェクトの対応                          | 差分                                                                                   |
| -------------------------------------------------------------------------------------- | --------------------------------------------- | -------------------------------------------------------------------------------------- |
| サブセット化してセルフホスティング（JIS第1水準・常用漢字・ひらがな・カタカナ・記号等） | Google Fonts（Noto Sans JP）を CDN で読み込み | **一部差分あり**：サブセット化・セルフホスティングは今回の検証範囲外。CDN で読み込み。 |

---

## 3. jQuery

| ルール                                                                       | 本プロジェクトの対応                                           | 差分                                                                                       |
| ---------------------------------------------------------------------------- | -------------------------------------------------------------- | ------------------------------------------------------------------------------------------ |
| 最新版を CDN で読み込み。WordPress 時は `jQuery(function ($) { });` 内に記述 | jQuery は使用していない。Vanilla JS（`assets/js/main.js`）のみ | **一部差分あり**：ルールは jQuery 前提。本プロジェクトは JS が軽いため Vanilla JS で実装。 |

---

## 4. CSS設計（FLOCSS・命名規則）

| ルール                                                                                              | 本プロジェクトの対応                                                                                                    | 差分     |
| --------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------- | -------- |
| FLOCSS ベース。レイアウトは `l-`、コンポーネントは `c-`、プロジェクトは `p-`、ユーティリティは `u-` | 同様のプレフィックスと BEM を採用（`l-header`, `l-footer`, `l-inner`, `c-button`, `p-top-kv`, `p-contact`, `u-ptb` 等） | **一致** |
| Element はアンダースコア2つ `__`。Modifier はダブルハイフン `--`。Element 内はハイフン `-`          | `block__element`, `block--modifier`, `block__element-item` の形式で統一                                                 | **一致** |
| ユーティリティには `!important` を付ける                                                            | `u-ptb`, `u-visually-hidden` 等で `!important` を使用                                                                   | **一致** |

---

## 5. id・要素・余白

| ルール                                                                 | 本プロジェクトの対応                                                                           | 差分                                                                             |
| ---------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------- |
| id にスタイルは当てない（詳細度を上げないため）                        | id に対するスタイル指定なし                                                                    | **一致**                                                                         |
| h1・p 等の要素に直接スタイルを当てず、クラスで指定                     | 本文・見出し等は `p-single-contents__text`, `p-single-contents__title-level2` 等のクラスで指定 | **一致**                                                                         |
| margin は上と左につける。Block には余白を持たせず、親の Element に余白 | 各所で上・左の margin や、ラッパー要素への余白を採用。統一の徹底度はファイルによりばらつきあり | **一部差分あり**                                                                 |
| セクション間の余白は `l-section` または各 `p-〇〇` に付与              | `u-ptb` や各 p- ブロックで余白を付与。`l-section` は未使用                                     | **一部差分あり**：`l-section` の共通クラスは使わず、ユーティリティや p- で対応。 |
| `l-inner` はコンテンツ幅と左右の padding（余白）を設定                 | `l-inner` で `width: 90%` + `margin: 0 auto` + `max-width` を設定。左右は margin で中央寄せ    | **一致**                                                                         |

## 6. ファイル構成

| ルール                                                                 | 本プロジェクトの対応                                                                                      | 差分                                                                             |
| ---------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------- |
| css / fonts / img / js は `assets` フォルダに格納                      | `assets/css`, `assets/img`, `assets/js` に格納。fonts は未使用                                            | **一致**                                                                         |
| SCSS で FLOCSS を基準にファイル分割し、1 つの `style.css` にコンパイル | `assets/scss` で foundation / layout / component / page / utility 等に分割。`assets/css/style.css` に出力 | **一致**                                                                         |
| `style.css` の次に空の `custom.css`（ディレクター調整用）を読み込み    | 全 HTML で `assets/css/style.css` の後に `assets/css/custom.css` を読み込み                               | **一致**                                                                         |
| Webフォントは `base.scss` で読み込み                                   | Webフォントは HTML の link で読み込み。SCSS では未読み込み                                                | **一部差分あり**：ルールでは base で読み込む想定。本プロジェクトは HTML で対応。 |

---

## 7. ブレイクポイント

| ルール                                                                    | 本プロジェクトの対応                                                         | 差分                                                                           |
| ------------------------------------------------------------------------- | ---------------------------------------------------------------------------- | ------------------------------------------------------------------------------ |
| 基本的に PC ファーストでコーディング                                      | メディアクエリは **min-width（モバイルファースト）** で記述                  | **一部差分あり**：ルールは PC ファースト。本プロジェクトはモバイルファースト。 |
| Bootstrap を参照した max-width のブレイクポイント（xxs〜xxl, header, pc） | `sm: 500px`, `md: 768px`, `lg: 1080px`, `xl: 1200px` の **min-width** で定義 | **一部差分あり**：値・キー名がルールと異なり、min-width ベース。               |

---

## 8. 画像

| ルール                                                                 | 本プロジェクトの対応                                                                | 差分           |
| ---------------------------------------------------------------------- | ----------------------------------------------------------------------------------- | -------------- |
| 基本的に 2 倍サイズで書き出し。TinyPNG 等で圧縮                        | 画像の書き出し倍率・圧縮は今回の検証では対応しておりません                          | **検証範囲外** |
| 写真は jpg、作図・透過は png、ロゴ・アイコンは svg                     | プロジェクト内で jpg / png / svg を用途に応じて使用                                 | **一致**       |
| img に **width / height** を指定（レイアウトシフト防止）               | 主要な img に width / height を指定                                                 | **一致**       |
| 遅延読み込みのため **loading="lazy"** を記述。メインビジュアル等は外す | 一覧・CTA・フッター等は `loading="lazy"`。KV やロゴ等ファーストビューは付けていない | **一致**       |
| PC/SP の画像切り替えは **picture** を使用（display: none 禁止）        | ロゴやバナー等で `<picture>` + `<source>` で切り替え                                | **一致**       |

## 11. その他（本プロジェクトで実施しているがルールに明示のないもの）

- **noindex**：全 HTML の `<head>` に `<meta name="robots" content="noindex" />` を記載（検索エンジンへのインデックス抑制）。
- **Sass**：`@mixin` の `arrow` は使用せず、必要なスタイルをコンポーネントに直接記述。`@extend` のプレースホルダー（%container）も使用せず、通常のクラスのみで記述。

---

## まとめ

- **合わせている点**：HTML5、FLOCSS 的な l-/c-/p-/u- と BEM、id にスタイルを当てない、要素ではなくクラスでスタイル、画像の width/height と loading="lazy" と picture、assets 配下のファイル構成、style.css + custom.css、コピーライトの動的表示。
- **意図的に違えている点**：Webフォントは CDN（サブセット・セルフホストなし）、jQuery 非使用（Vanilla JS）、ブレイクポイントは min-width・モバイルファースト、Webフォント読み込みは HTML で実施。
- **今回の検証範囲外**：フォントサブセット、画像の2倍書き出し・圧縮のルール化、PageSpeed/Lighthouse のスコア検証、W3C バリデータによるチェックについては、今回の検証範囲外としております。
